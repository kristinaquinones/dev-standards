## Test Structure

### Introduction (5 minutes)

- Welcome and explain test purpose
- Explain think-aloud process
- Clarify you're testing the design, not them
- Get consent if recording

### Tasks (20-30 minutes)

- Present tasks one at a time
- Observe without helping
- Ask follow-up questions after each task
- Take notes on problems and observations

### Wrap-up (5-10 minutes)

- Ask overall impressions
- Ask what they liked/disliked
- Ask for suggestions
- Thank them for participating

## Example

**Test:** Task Filtering Usability Test  
**Goals:** Can users find and use the filter feature? Is it intuitive?

**Tasks:**
1. Find tasks assigned to a specific team member
2. Clear the filter to see all tasks again
3. Use filter with a different team member

**Metrics:**
- Task completion rate
- Time to complete each task
- Number of errors or confusion points
- User satisfaction rating

**Issues to Watch:**
- Can users find the filter dropdown?
- Is it clear how to use it?
- Is the filtered state obvious?
- Can users clear the filter easily?
